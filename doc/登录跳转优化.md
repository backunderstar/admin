# 登录后跳转优化：消除 Vue Router 警告

> 解决登录成功后 `router.push()` 触发 `[Vue Router warn]: No match found` 的问题。

---

## 一、问题现象

用户登录成功后，控制台出现警告：

```
[Vue Router warn]: No match found for location with path "/template/cards"
```

无论跳转目标是什么路径（包括 `/`），只要在登录后立即调用 `router.push()`，都会触发此警告。

---

## 二、根因分析

### 2.1 路由注册的时序问题

本系统采用**双阶段路由注册**（参见 [路由实现.md](./路由实现.md)）：

```
阶段一（应用启动）：注册静态路由 → /login、/:pathMatch(.*)*
阶段二（登录后）：  注册动态路由 → /dashboard/*、/template/*、/system/* 等
```

### 2.2 登录跳转的执行流程

原始代码在登录成功后直接调用 `router.push(redirect)`：

```ts
// LoginView.vue — 登录成功后
const redirect = (route.query.redirect as string) || '/'
router.push(redirect) // ❌ 此时动态路由尚未注册
```

此时路由器中**只有静态路由**（`/login` 和 catch-all 404），目标路径如 `/template/cards` 还未注册。Vue Router 在路径解析阶段找不到匹配的路由记录，即使命中了 catch-all，仍会输出警告。

### 2.3 为什么连 `/` 也会警告

`/` 并不是一个已注册的静态路由。静态路由只有：

| 路径               | 说明          |
| ------------------ | ------------- |
| `/login`           | 登录页        |
| `/:pathMatch(.*)*` | catch-all 404 |

`/` 属于动态路由的一部分（布局容器路由），需要在阶段二注册后才存在。

### 2.4 问题本质

**在动态路由注册之前，无法安全地调用 `router.push()` 跳转到任何业务页面。**

---

## 三、解决方案

### 3.1 核心思路

```
登录成功 → 存储目标路径 → 通过已注册的 /login 路由触发守卫
         → 守卫注册动态路由 → 读取存储的路径 → 执行实际跳转
```

**关键原则：永远不 `router.push()` 到未注册的路径。**

### 3.2 实现细节

#### LoginView.vue — 登录成功后

```ts
// 1. 确定目标路径
const redirect = (route.query.redirect as string) || '/'

// 2. 存入 sessionStorage（跨导航持久化）
sessionStorage.setItem('login_redirect', redirect)

// 3. 通过已注册的 /login 路由触发守卫
//    使用 _rd 时间戳参数让 Vue Router 将其视为新导航（同路径 + 不同 query = 新位置）
router.replace({ name: 'Login', query: { _rd: String(Date.now()) } })
```

**为什么用 `router.replace` 而不是 `router.push`：** 避免在历史记录中留下多余的 `/login` 条目。

**为什么需要 `_rd` 时间戳参数：** Vue Router 会忽略到相同路由的重复导航。添加动态参数使其成为"不同"的目标，从而触发守卫。

#### beforeEach.ts — 守卫中处理跳转

```ts
// 动态路由注册完成后、静态路由检查之前

// 8. 读取登录页通过 sessionStorage 传递的 redirect 路径
const loginRedirect = sessionStorage.getItem('login_redirect')
if (loginRedirect) {
  sessionStorage.removeItem('login_redirect')  // 读后即删，防止残留
  routeInitInProgress = false

  // 验证目标路径权限
  const { homePath } = useCommon()
  const { path: validatedPath, hasPermission } = RoutePermissionValidator.validatePath(
    loginRedirect,
    menuList,
    homePath.value || '/',
  )

  closeLoading()
  return {
    path: hasPermission ? loginRedirect : validatedPath,
    replace: true,
  }
}

// 9. 静态路由检查（在 redirect 之后，避免 /login 被拦截）
if (isStaticRoute(to.path)) { ... }
```

**为什么 redirect 检查必须在静态路由检查之前：** 守卫被触发时 `to.path` 是 `/login`（静态路由），如果先执行静态路由检查会直接 return，永远读不到 sessionStorage 中的 redirect 路径。

## 四、完整流程图

### 4.1 从受保护页面被重定向登录

```
用户访问 /template/cards
  │
  ▼
beforeEach: 未登录 → 重定向到 /login?redirect=/template/cards
  │
  ▼
用户登录成功
  │
  ▼
LoginView: sessionStorage.setItem('login_redirect', '/template/cards')
LoginView: router.replace({ name: 'Login', query: { _rd: 1718xxx } })
  │
  ▼
beforeEach: to = /login?_rd=1718xxx
  ├─ 登录检查 → 通过（已登录）
  ├─ 动态路由未注册 → handleDynamicRoutes()
  │   ├─ 获取用户信息
  │   ├─ 获取菜单数据
  │   ├─ 注册动态路由 ✓
  │   ├─ 读取 sessionStorage → '/template/cards' ✓
  │   ├─ 权限验证 → 通过
  │   └─ return { path: '/template/cards', replace: true }
  │
  ▼
beforeEach: to = /template/cards（路由已注册，matched.length > 0）
  ├─ 登录检查 → 通过
  ├─ 动态路由已注册 → 跳过
  ├─ matched.length > 0 → setWorktab + setPageTitle
  └─ 正常渲染页面 ✓
```

### 4.2 直接访问登录页

```
用户直接访问 /login
  │
  ▼
用户登录成功
  │
  ▼
LoginView: route.query.redirect 不存在 → redirect = '/'
LoginView: sessionStorage.setItem('login_redirect', '/')
LoginView: router.replace({ name: 'Login', query: { _rd: 1718xxx } })
  │
  ▼
beforeEach: handleDynamicRoutes()
  ├─ 注册动态路由
  ├─ 读取 sessionStorage → '/'
  ├─ 权限验证 → 通过
  └─ return { path: '/', replace: true }
  │
  ▼
beforeEach: to = /
  ├─ handleRootPathRedirect() → 重定向到 homePath（如 /dashboard/console）
  └─ 正常渲染首页 ✓
```

## 五、涉及的文件变更

| 文件                              | 变更内容                                                             |
| --------------------------------- | -------------------------------------------------------------------- |
| `src/views/LoginView.vue`         | 登录成功后改为 sessionStorage 存储 + `router.replace` 触发守卫       |
| `src/router/guards/beforeEach.ts` | 动态路由注册后优先检查 sessionStorage redirect，置于静态路由检查之前 |

---

## 六、注意事项

1. **sessionStorage 而非 localStorage：** redirect 路径仅在当前标签页有效，关闭标签页自动清除，不会跨标签页污染。

2. **读后即删：** 守卫读取 `login_redirect` 后立即 `removeItem`，防止刷新页面时重复跳转。

3. **`_rd` 时间戳参数：** 仅用于触发守卫，不影响业务逻辑。守卫重新导航时使用 `replace: true`，不会残留在历史记录中。

4. **权限验证：** 即使是 redirect 路径，也会经过 `RoutePermissionValidator.validatePath()` 验证。无权限时自动降级到首页。
