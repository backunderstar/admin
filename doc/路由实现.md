# 路由实现

## 概述

系统路由采用**双阶段注册**策略，将路由分为静态路由和动态路由两部分。

## 核心文件

```
src/router/
├── index.ts                      # 创建路由实例，导出路由和初始化方法
├── guards/
│   ├── beforeEach.ts             # 前置守卫：登录验证 + 动态路由注册 + 权限校验
│   └── afterEach.ts              # 后置守卫：页面标题、进度条、工作标签页
├── routes/
│   ├── asyncRoutes.ts            # 异步路由模板（当前为空，已迁移到 modules/）
│   └── staticRoutes.ts           # 静态路由定义
├── modules/
│   ├── index.ts                  # 路由模块聚合入口，统一导出 routeModules
│   ├── dashboard.ts              # 仪表盘模块
│   └── exception.ts              # 异常页面模块
└── core/
    ├── RouteRegistry.ts          # 路由注册器
    ├── ComponentLoader.ts        # 组件加载器（字符串路径 → Vue 组件）
    ├── RouteTransformer.ts       # 路由转换器
    ├── MenuProcessor.ts          # 菜单处理器
    ├── RoutePermissionValidator.ts # 路由权限验证器
    └── IframeRouteManager.ts     # Iframe 路由管理
```

## 新建子路由页面指南

### 第一步：创建视图文件

在 `src/views/` 下创建对应的 `.vue` 文件。路径结构与路由 `component` 字段对应。

**路径映射规则**（由 `ComponentLoader` 处理）：

| 路由 component 值      | 实际解析路径                                       |
| ---------------------- | -------------------------------------------------- |
| `'/dashboard/console'` | `src/views/dashboard/console.vue`                  |
| `'/system/user'`       | `src/views/system/user.vue`                        |
| `'/form'`              | `src/views/form.vue` 或 `src/views/form/index.vue` |
| `'/index'`             | `src/views/index.vue`（这是 Layout 布局壳组件）    |

> **注意**：`component` 字段是**字符串**，以 `/` 开头，**不带** `.vue` 后缀，相对于 `src/views/` 目录。

### 第二步：定义路由模块

在 `src/router/modules/` 下创建路由模块文件。每个模块导出一个 `AppRouteRecord` 对象。

**完整示例 — 假设新建一个「系统管理」模块：**

```typescript
// src/router/modules/system.ts
import type { AppRouteRecord } from '@/types/router'

export const systemRoutes: AppRouteRecord = {
  name: 'System', // 唯一路由名称
  path: '/system', // 唯一路径前缀
  component: '/index', // ⚠️ 父级固定用 '/index'（即 Layout 壳组件）
  meta: {
    title: 'menus.system.title', // i18n 翻译 key
    icon: 'ri:settings-3-line', // Iconify 图标名
    roles: ['R_SUPER', 'R_ADMIN'], // 可访问的角色（不填则所有角色可访问）
  },
  children: [
    {
      path: 'user', // 子路由不用加前缀，最终路径 = /system/user
      name: 'SystemUser', // 全局唯一的路由名称
      component: '/system/user', // → src/views/system/user.vue
      meta: {
        title: 'menus.system.user',
        icon: 'ri:user-line',
        keepAlive: true, // 是否缓存组件
      },
    },
    {
      path: 'role',
      name: 'SystemRole',
      component: '/system/role',
      meta: {
        title: 'menus.system.role',
        icon: 'ri:user-settings-line',
        roles: ['R_SUPER'], // 子路由可以进一步限制角色
      },
    },
  ],
}
```

**要点：**

- 父级路由的 `component` 固定写 `'/index'`，它对应 Layout 壳组件（侧边栏 + 顶栏 + 内容区）
- 子路由的 `component` 写实际页面路径，如 `'/system/user'`
- `path` 在子路由中是相对路径，最终拼接为 `/system/user`
- `name` 必须全局唯一，建议用 `模块名+页面名` 的 PascalCase 格式

### 第三步：注册路由模块

在 `src/router/modules/index.ts` 中导入并添加到 `routeModules` 数组：

```typescript
import type { AppRouteRecord } from '@/types/router'
import { dashboardRoutes } from './dashboard'
import { exceptionRoutes } from './exception'
import { systemRoutes } from './system' // ← 新增

export const routeModules: AppRouteRecord[] = [
  dashboardRoutes,
  exceptionRoutes,
  systemRoutes, // ← 新增
]
```

### 第四步：添加 i18n 翻译

在 `src/locales/langs/zh.json` 和 `en.json` 的 `menus` 字段下添加对应翻译：

```jsonc
// zh.json → menus
"system": {
  "title": "系统管理",
  "user": "用户管理",
  "role": "角色管理"
}
```

翻译 key 与路由 meta.title 对应，如 `menus.system.user` → `menus.system` 下的 `user` 字段。

### 第五步：验证

```bash
pnpm dev
```

登录后侧边栏应自动显示新模块菜单。如果菜单未出现，检查：

1. `roles` 配置是否包含当前登录角色
2. `meta.title` 的 i18n key 是否正确
3. 组件文件路径是否与 `component` 字段匹配

## 路由 Meta 属性速查

| 属性            | 类型       | 说明                                                           |
| --------------- | ---------- | -------------------------------------------------------------- |
| `title`         | `string`   | **必填**，菜单/标签页标题，支持 i18n key                       |
| `icon`          | `string`   | 菜单图标，Iconify 图标名（如 `ri:home-line`）                  |
| `roles`         | `string[]` | 可访问的角色列表，如 `['R_SUPER', 'R_ADMIN']`。不填 = 所有角色 |
| `keepAlive`     | `boolean`  | 是否缓存组件（默认 false）                                     |
| `isHide`        | `boolean`  | 在侧边栏菜单中隐藏（但仍可访问）                               |
| `isHideTab`     | `boolean`  | 在顶部标签页中隐藏                                             |
| `fixedTab`      | `boolean`  | 固定标签页，不可关闭                                           |
| `isFullPage`    | `boolean`  | 全屏页面（不显示 Layout 壳）                                   |
| `link`          | `string`   | 外部链接 URL（点击后新窗口打开）                               |
| `isIframe`      | `boolean`  | 是否以 iframe 方式嵌入外部页面                                 |
| `authList`      | `Array`    | 后端权限模式下的操作权限列表                                   |
| `activePath`    | `string`   | 高亮的菜单路径（用于详情页等场景）                             |
| `showBadge`     | `boolean`  | 菜单项显示红点徽章                                             |
| `showTextBadge` | `string`   | 菜单项显示文本徽章（如 "Hot"、"New"）                          |

## 路由定义类型

```typescript
interface AppRouteRecord {
  name?: string // 路由名称（唯一）
  path: string // 路由路径
  component?: string // 组件路径（字符串，如 '/dashboard/console'）
  meta: RouteMeta // 路由元数据
  children?: AppRouteRecord[] // 子路由
}
```

`component` 字段为字符串，由 `ComponentLoader` 在路由注册时解析为实际的 Vue 组件。父级路由固定使用 `'/index'`（Layout 壳），子路由使用具体页面路径。

## 组件路径解析规则

`ComponentLoader` 使用 `import.meta.glob('../../views/**/*.vue')` 扫描所有视图文件，解析逻辑：

```
component: '/dashboard/console'
  → 尝试 ../../views/dashboard/console.vue  ✓ 命中
  → 尝试 ../../views/dashboard/console/index.vue  （备选）

component: '/index'
  → ../../views/index.vue  ✓（Layout 壳组件）
```

## 静态路由 vs 动态路由

### 静态路由（`src/routes/staticRoutes.ts`）

无需权限，应用启动时立即注册：

- `/login` — 登录页
- `/:pathMatch(.*)*` — 404 兜底页

### 动态路由（`src/router/modules/*.ts`）

需要登录后根据用户权限动态注册：

- `modules/index.ts` 聚合所有模块路由
- 登录成功后由 `beforeEach` 守卫调用 `RouteRegistry.register()` 完成注册

## 路由注册流程

```
用户登录
  ↓
beforeEach 守卫检测到未初始化
  ↓
fetchGetUserInfo() → 获取用户信息和角色
  ↓
MenuProcessor.getMenuList() → 获取菜单列表
  ↓
RouteRegistry.register(menuList)
  ├── ComponentLoader.load() → 解析组件路径
  ├── RouteTransformer.transform() → 转换路由配置
  └── router.addRoute() → 逐条注册到 Vue Router
  ↓
RoutePermissionValidator.validatePath() → 验证目标路径权限
  ↓
放行导航 or 重定向到首页/403
```

## 菜单生成

侧边栏菜单**从路由配置自动生成**，不需要单独维护菜单数据：

1. 遍历 `routeModules` 中所有路由
2. 过滤掉 `meta.isHide === true` 的路由
3. 根据 `meta.roles` 过滤当前用户无权访问的菜单
4. 提取 `meta.title`、`meta.icon`、子路由等信息生成菜单树
5. 由 `RecursiveMenuItem.vue` 组件递归渲染菜单

## 权限控制

### 前端权限模式（`VITE_ACCESS_MODE=frontend`）

- 路由级别：`meta.roles` 数组控制（如 `['R_SUPER']`）
- 按钮级别：`useAuth().hasAuth(authMark)` 判断

### 后端权限模式（`VITE_ACCESS_MODE=backend`）

- 路由级别：后端返回的菜单列表决定哪些路由可注册
- 按钮级别：`meta.authList` 配合后端返回的权限标识判断

## 面包屑导航

面包屑根据当前路由的**匹配记录**（`route.matched`）自动生成，每一级路由对应一级面包屑，无需额外配置。

## KeepAlive 缓存

路由的 `meta.keepAlive` 控制是否缓存该页面组件：

- `keepAlive: true` → 页面切换时保留组件状态（表单数据、滚动位置等）
- `keepAlive: false`（默认）→ 每次进入重新创建组件
